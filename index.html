Everyday Git

What is Git?

Definition

GIT isa version control system used to track changes in source code during software development. It allows multiple developers to collaborate on a project by enabling them to track and merge changes efficiently.

Git as a version control system

A version control system helps manage changes to files and directories over time.Some other popular alternatives version control systems to GIT include:

  1. Subversion(SVN)
  2. Mercurial
However GIT is widely regarded as the most popular due to its flexibility, speed, and disturbed nature. In this blog we`ll focus on GIT and GitHub (a platform that uses GIT).

How to install Git

Before the installation, it's important to confirm whether you have git installed on your computer. You do this by running the code "git --version". If you notice a version number, then you have it installed. There are also a few ways to install Git on Windows. The most official build is available for download on the Git website. Just go to https://git-scm.com/download/win and the download will start automatically. Note that this is a project called Git for Windows, which is separate from Git itself; for more information on it, go to https://gitforwindows.org. To get an automated installation you can use the Git Chocolatey package. Note that the Chocolatey package is community maintained.

Configuring Git

To configure git usename and email, you type the following commands:

  1. git config --global user.name "name"
  2. git config --global user.email "email"
  3. To confirm configuration, use the code "git config --list"

Initializing Git Repository

Initializing a Git repository is a simple process. First, open a terminal and navigate to your project’s directory. Next, run the command git init. This creates a hidden .git folder that stores your repository’s history, configuration, and metadata.

Branches

A branch is a movable pointer to a specific commit that represents an independent line of development. It allows you to isolate work on features, fixes, or experiments without impacting the main codebase. When you create a new branch, Git simply creates a new pointer that starts at your current commit. As you make commits on that branch, the pointer advances, building a separate history. This flexibility makes branching essential for parallel development, enabling teams to work on multiple tasks simultaneously and merge changes later once they’re stable and reviewed.

Creating a branch

Creating a branch in Git is simple. Use the command “git branch ” to generate a new branch pointer starting at your current commit. For convenience, you can combine branch creation and checkout using “git checkout -b ”, immediately switching your working directory to the new branch. It simplifies collaboration.

Viewing branches

To view all branches in Git, use the command “git branch” for local branches or” git branch -a” to include remote branches. This command lists every branch in your repository, showing the current active branch clearly highlighted. It helps you track different lines of development and manage collaborative workflows effectively.

Switching Branches

To switch branches in Git, use the command: “git checkout ”. This command moves you to the specified branch, updating your working directory to reflect its state. Before switching, ensure you commit or stash any changes to avoid conflicts or lost work.

Merging Branches

To merge branches, first switch to the branch you want to merge changes into (typically the main branch). Then, execute: “git merge ”. Git combines the histories. If conflicts occur, manually resolve them before completing the merge.

Deleting branches

To delete a branch locally in Git, use: • Safe Deletion: “git branch -d ”. This deletes the branch only if it’s fully merged. • Force Deletion: “git branch -D ”. Use this to force delete an unmerged branch. For remote branches, execute: “git push origin --delete ”. This command removes the branch from the remote repository. Always ensure you’ve backed up any important work before deleting a branch.

Recommendation

After mastering local Git operations, the next step is to delve into remote repository management and collaboration tools. Start by learning how to push, pull, and fetch changes from remote repositories, and then connect your local repository to platforms like GitHub, GitLab, or Bitbucket. This will enable you to manage pull requests, fork repositories, and contribute to open-source projects. Additionally, exploring advanced Git techniques-such as interactive rebasing, cherry-picking, and conflict resolution—will further refine your workflow and improve your version control skills, ultimately paving the way for more efficient team collaboration and project management.
